Installation

Global (all projects)

git clone https://github.com/TooFaded420/torch.git \
  ~/.claude/skills/torch
cd ~/.claude/skills/torch && ./setup

Per-Project (vendored)

cp -Rf ~/.claude/skills/torch \
  .claude/skills/torch
rm -rf .claude/skills/torch/.git
cd .claude/skills/torch && ./setup

Multi-Agent (Codex, Cursor, Gemini)

git clone https://github.com/TooFaded420/torch.git \
  ~/torch
cd ~/torch && ./setup --host auto

🧭
Planning
Plan
  • /office-hours Product reframing — 6 forcing questions, 3 implementation approaches with effort estimates. Best entry point for new ideas.
  • /plan-ceo-review Scope management — 4 modes: expand, selective expand, hold, reduce. Finds the "10-star product" hiding in your request.
  • /plan-eng-review Architecture lock-in — ASCII diagrams, edge cases, test matrix, security concerns, failure mode analysis.
🎨
Design
Design
  • /plan-design-review Pre-build design audit — rates dimensions 0-10, detects "AI slop", asks one question per design choice.
  • /design-consultation Design system creation — competitive research, safe + creative-risk options, generates DESIGN.md artifact.
  • /design-review Post-build design audit — same dimension audit, fixes issues, before/after screenshots, atomic commits.
🔍
Review
Review
  • /review Code quality gate — finds production bugs passing CI, auto-fixes obvious issues, flags edge cases. Atomic commits.
  • /codex Multi-model second opinion via OpenAI Codex — 3 modes: pass/fail gate, adversarial challenge, consultation.
  • /autoplan Auto-review pipeline — runs CEO, design, and eng reviews sequentially with auto-decisions. Surfaces taste decisions at a final gate.
  • /investigate Systematic debugging — traces data flow, tests hypotheses. "Iron Law": no fix without investigation. Stops after 3 fails.
🧪
Testing
Test
  • /qa Full QA — opens real Chromium, clicks through UI, screenshots bugs, auto-generates regression tests, fixes & re-verifies.
  • /qa-only Report-only QA — same testing methodology but no code changes. Great for documentation or handoff.
  • /benchmark Performance regression detection — baselines page load, Core Web Vitals, bundle sizes. Compares before/after on every PR.
  • /browse Browser automation — navigate, fill forms, screenshot. Use $B handoff for CAPTCHAs, $B resume to continue.
🚀
Ship & Docs
Ship
  • /ship Full release — sync main, run tests, coverage audit, bootstrap test frameworks if missing, push branch, open GitHub PR.
  • /land-and-deploy Post-PR workflow — merges the PR, waits for CI and deploy, verifies production health via canary checks. Picks up after /ship.
  • /canary Post-deploy monitoring — watches live app for console errors, perf regressions, page failures. Periodic screenshots + baseline diffs.
  • /document-release Doc sync — updates README, ARCHITECTURE, CONTRIBUTING, CLAUDE.md, TODOS. Auto-invoked by /ship.
  • /retro Weekly retrospective — team contributions, shipping streaks, velocity, test health, coverage trends. Now supports /retro global for cross-project reports.
🛡
Safety & Guards
Safety
  • /cso Full security audit — OWASP Top 10, STRIDE threat model, attack surface analysis, auth flow verification, dependency CVE scan, supply chain risk.
  • /careful Warns before destructive ops (rm -rf, DROP TABLE, force-push, git reset --hard). Activate with "be careful".
  • /freeze Lock editing to a single directory. Prevents accidental changes outside debug scope.
  • /guard Combined /careful + /freeze. Maximum safety for production environments.
  • /unfreeze Remove /freeze restrictions, restore full editing capability.
⚙
Utilities
Utility
  • /torch-upgrade Self-updater — detects global vs. vendored installs, syncs both to latest, shows changelog.
  • /setup-browser-cookies Import cookies from Chrome/Arc/Brave/Edge for testing authenticated pages.

Add to Your Project's CLAUDE.md

Paste this into every project's CLAUDE.md so Claude Code discovers torch skills:

## torch
Use /browse from torch for all web browsing. Never use mcp__claude-in-chrome__* tools.

Available skills: /office-hours, /plan-ceo-review, /plan-eng-review,
/plan-design-review, /design-consultation, /autoplan, /review, /ship,
/land-and-deploy, /canary, /benchmark, /browse, /qa, /qa-only, /design-review,
/setup-browser-cookies, /retro, /investigate, /document-release, /codex, /cso,
/careful, /freeze, /guard, /unfreeze, /torch-upgrade
Global = All Projects
Install once to ~/.claude/skills/torch and every project gets all 21 skills automatically. Still add the CLAUDE.md section to each project.
Vendor for Teammates
Copy into .claude/skills/torch within the repo and commit it. Everyone on the team gets the same skills without installing globally.
Parallel Sprints
Run 10-15 concurrent Claude Code sessions. Each can be at a different pipeline stage — one planning, one building, one in QA.
Auto-Upgrade
Set auto_upgrade: true in ~/.torch/config.yaml to stay on the latest version without running /torch-upgrade manually.
Debug Safely
Start with /guard to lock edits to one directory + warn on destructive ops. Use /investigate for systematic root-cause analysis.
Requirements
Claude Code + Git + Bun v1.0+. For /codex, also install OpenAI's Codex CLI separately.
